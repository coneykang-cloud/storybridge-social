## 1. 브랜드 아이덴티티

### 브랜드 컨셉 (변경 없음)
> **"따뜻하고 안전한 이야기 공간"**

### v2 추가 — 3-Track 아이덴티티

각 역할(Track)마다 고유한 색상 정체성을 부여하여 생성 주체를 직관적으로 구분한다.

| Track | 역할 | 색상 | 아이콘 | 설명 |
|---|---|---|---|---|
| **Track A** | 치료사 주도 | Lavender `#C9B8E8` | 🩺 | 임상·치료 맥락 |
| **Track B** | 보호자 주도 | Soft Mint `#A8D8D8` | 👩 | 일상·가정 맥락 |
| **Track C** | 교사 주도 | Peach `#F5C5A3` | 👩‍🏫 | 학교·교실 맥락 |
| **아이 역할** | 읽기 전용 | Soft Yellow `#FEF9C3` | 🧒 | 소셜 스토리 열람 전용 — v2.5 신규 |

### v2 추가 — 연구 기반 UI 원칙
> **"혼합 청킹 + 누적 제시"** 조합을 UI 전반에서 최우선 추천 조합으로 시각적으로 강조한다.  
> 연구 데이터(EF M=80.64)를 근거로 ⭐ 아이콘 + 추천 문구를 고정 표시.

### 4.9 EditableStoryTitle — NEW v2.3

**기본 상태 (생성자 본인 열람 시):**
```
급식 줄 서기 ✏️
```

**편집 상태 (연필 아이콘 클릭 시):**
```
[급식 줄 서기 줄서기 연습      ] ✓ ✕
```

**Props:** `{ storyId: string, title: string, editable: boolean }`

**동작:**
- `editable`은 호출부에서 `creator_id === session.user.id` 비교 결과를 전달 (생성자 본인에게만 연필 아이콘 노출)
- 평소엔 일반 텍스트로 렌더링되며, 제목에 마우스를 올리면 `text-soft-gray` 연필 아이콘이 페이드인
- 클릭 시 같은 자리가 `<input>`으로 전환 (`text-lg font-semibold border-b border-mint-400 focus:outline-none`)
- `Enter` 또는 input `blur` → `PATCH /api/story/[id]` with `{ title }` 호출, 성공 시 즉시 반영 (별도 새로고침 없음)
- `Escape` 또는 ✕ 클릭 → 변경 취소하고 원래 제목으로 복귀
- 저장 실패 시 토스트로 에러 안내, 입력값은 유지하여 재시도 가능

**스타일:**
- 연필 아이콘: `Tabler IconPencil`, `w-4 h-4 text-soft-gray opacity-0 group-hover:opacity-100 transition-opacity`
- 저장(✓)/취소(✕) 아이콘: `text-mint-500` / `text-coral-400`, 클릭 영역 `p-1 rounded hover:bg-gray-100`

> **본문은 인라인 수정 대상이 아니다.** 본문 내용 변경은 자문 댓글 → 수정 제안 플로우(추후 구현 예정, PRD §9 "스토리 본문 수정" 행 참조)로 분리되어 있으며, EditableStoryTitle은 제목에만 적용된다.

### 4.10 DeleteStoryButton — NEW v2.3

```
┌──────────┐
│ 🗑 삭제  │
└──────────┘
```

**Props:** `{ storyId: string }`

**노출 조건:** `isParent || isCreator`
- `isParent`: 로그인 사용자가 스토리 대상 아동(`story.children.parent_id`)의 보호자인 경우 — 자녀의 모든 스토리 삭제 가능
- `isCreator`: 로그인 사용자가 해당 스토리의 `creator_id` 본인인 경우 — 본인이 만든 스토리는 본인이 삭제 가능 (치료사·교사 포함)

> 두 조건 중 하나만 충족해도 노출되며, 클라이언트의 노출 조건은 서버(`DELETE /api/story/[id]`)와 RLS 정책(마이그레이션 013) 양쪽에 동일하게 반영되어야 한다 — 한쪽만 맞으면 "성공했다는데 안 지워지는" RLS 침묵 실패가 재현된다.

**동작:**
- 클릭 → 확인 다이얼로그 ("이 스토리를 삭제할까요? 되돌릴 수 없어요") → 확인 시 `DELETE /api/story/[id]` 호출
- 성공 시 Storage에 저장된 스토리 이미지까지 함께 정리되고, 목록/책장으로 리다이렉트

### 4.14 DiffViewer — NEW v2.6 (LCS 기반 단어 단위 하이라이트)

```
변경 전                          변경 후
┌────────────────────────┐      ┌────────────────────────┐
│ 줄 맨 뒤에 서서 내      │      │ 줄 맨 뒤에 서서 내      │
│ 차례를 ~~기다려요~~.   │      │ 차례를 ▒조용히▒        │
│                        │      │ ▒기다려요▒.            │
└────────────────────────┘      └────────────────────────┘
   ~~취소선~~ = 삭제된 단어         ▒음영▒ = 추가된 단어
```

**Props:** `{ before: string, after: string }`

**동작:**
- `tokenize(text)`: `text.split(/(\s+)/)`로 공백을 보존하며 단어 단위 토큰화
- `diffTokens(beforeTokens, afterTokens)`: LCS(최장 공통 부분열) DP 테이블을 계산해 각 토큰을 `equal`/`remove`/`add`로 분류
- 좌측(변경 전) 컬럼에는 `equal` + `remove` 토큰만, 우측(변경 후) 컬럼에는 `equal` + `add` 토큰만 렌더링 — 문단 전체가 아닌 **달라진 단어/구절만** 하이라이트되어 어디가 바뀌었는지 한눈에 파악 가능 (이전엔 두 문단을 통째로 나란히 보여주기만 했음)

**스타일:**
- 컨테이너: `grid grid-cols-2 gap-3`
- 컬럼 헤더: `text-xs text-soft-gray font-medium mb-1` ("변경 전" / "변경 후")
- 컬럼 박스: `bg-gray-50 rounded-xl p-3 text-sm leading-relaxed`
- 삭제된 토큰(`remove`): `text-coral-500 line-through bg-coral-50 rounded px-0.5`
- 추가된 토큰(`add`): `bg-warning-amber/30 text-charcoal font-medium rounded px-0.5`
- 변경 없는 토큰(`equal`): 기본 텍스트 스타일
