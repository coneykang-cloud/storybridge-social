## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| **제품명** | StoryBridge (스토리브릿지) |
| **슬로건** | "우리 아이의 이야기, 함께 만들어요" |
| **버전** | MVP v2.0 |
| **플랫폼** | 반응형 Web App (Mobile-first, 웹 전용) |
| **언어** | 한국어 전용 |
| **참조 서비스** | Pictello (AssistiveWare), Social Story Creator App, YesChat.ai |

### 핵심 가치 제안

- **초개인화 아바타:** 아이 사진 1장 → AI 아바타(지브리풍 / 사진닮은꼴 등) 즉시 생성
- **AI 자동 스토리 생성:** Carol Gray Social Stories™ 10.4 기준 + 설명문·조망문·지시문 구조를 AI가 자동 적용, YesChat.ai 방식의 LLM 기반 실시간 생성·인라인 편집
- **3-Track 생성 모델:** 치료사·보호자·교사가 각자의 전문 맥락에서 스토리를 직접 생성하고, 역할 기반 협업·승인 플로우로 품질 보증
- **스토리 Pool:** 연령대별·카테고리별 사전 제작 템플릿 제공 (학교생활·또래관계 / 일상생활·감정조절)

## 2. 배경 및 근거

### 문제 정의

자폐 스펙트럼 장애(ASD) 아동은 사회적 상황에서의 맥락 이해와 행동 전이에 어려움을 겪는다. 기존 소셜 스토리는 다음 세 가지 구조적 한계를 지닌다.

1. **시각적 자아 일치도 부재:** 일반화된 캐릭터 사용으로 아동의 자기 참조 효과(Self-Reference Effect) 유도 불가
2. **인지적 과부하:** 평면적 정보 나열로 작업 기억(Working Memory) 용량이 부족한 ASD 아동에게 과부하 발생
3. **제작 현실성 한계:** 보호자·치료사·교사가 개별 상황에 맞는 스토리를 매번 수작업으로 제작하는 것은 시간·비용 면에서 불가능에 가까움

### 이론적 근거

| 이론 | 적용 |
|---|---|
| 인지부하이론 (Sweller, 1988) | 시공간적 청킹으로 외재적 인지부하 감소 |
| 상황 모델 이론 (Zwaan & Radvansky, 1998) | 시간·공간 단서 명시화로 상황 몰입 지원 |
| 자기 참조 효과 (Rogers et al., 1977) | AI 아바타·맞춤 상황으로 자기 관련성 극대화 |
| Social Stories™ 10.4 (Gray, 2023) | 10가지 기준을 AI 프롬프트 파이프라인에 반영 |
| 실행 기능 이론 (Miyake et al., 2000) | 청킹 기반 외부 비계로 계획 수립·억제 통제 지원 |

## 3. 타겟 사용자 (페르소나)

### 사용자 그룹

| 페르소나 | 연령/역할 | 주요 니즈 | 핵심 페인 포인트 |
|---|---|---|---|
| 🧒 **아동** | 7~18세 ASD 진단 | 나와 닮은 주인공, 큰 버튼, 음성 읽기 | 일반 캐릭터로 몰입 불가, 정보 과부하 |
| 👩 **보호자** | 30~50대 부모/양육자 | 빠른 스토리 생성, 전문가 소통, 최종 승인권 | 매번 수작업 제작, 치료사와 소통 단절 |
| 🩺 **치료사** | 언어·행동치료사 (SLP/ABA) | 다수 아동 스토리 직접 생성·관리, 자문 제공 | 아동별 맞춤 스토리 제작 시간 과다 |
| 👩‍🏫 **교사** | 특수교사·일반교사 | 학교 맥락 스토리 직접 생성, 가정 연계 | 가정과의 스토리 연계 어려움, 개별 제작 부담 |

### 연령대별 아동 세그먼트

| 세그먼트 | 연령 | 특징 | UI 고려사항 |
|---|---|---|---|
| 저연령 | 7~9세 | 짧은 문장, 단순 상황 | 아주 큰 버튼, 그림 중심, 1문장/페이지 |
| 중연령 | 10~13세 | 또래 관계·감정 복잡화 | 보통 버튼, 그림+텍스트 균형 |
| 고연령 | 14~18세 | 학교생활·자기 이해 | 텍스트 비중 증가, 성숙한 디자인 |

## 5. 핵심 기능 상세

### F1. 아바타 스튜디오

- 사진 업로드 (JPEG / PNG, 얼굴 1인 기준)
- **스타일 4종:**
  - 🎨 **지브리풍** — 스튜디오 지브리 애니메이션 스타일
  - 🪞 **사진닮은꼴** — 아이의 실제 외모를 반영한 리얼리스틱 캐릭터
  - 🎬 **픽사풍** — 픽사/디즈니 스타일의 3D 캐릭터
  - 🖌️ **수채화** — 부드러운 수채화 일러스트 스타일
- DALL·E 3 API 연동, 생성 후 라이브러리 저장
- 아동 프로필당 아바타 최대 5개 저장
- 생성된 아바타는 모든 스토리에서 재사용 가능

### F2. 스토리 Pool (템플릿 라이브러리)

연령대(3구간) × 카테고리(2종) 교차 구성으로 총 **6개 섹션**의 사전 제작 템플릿 제공.

각 템플릿은 아동 프로필 반영 후 AI가 즉시 커스터마이징. **보호자·치료사·교사 모두 사용 가능.**

#### 카테고리 A — 학교생활·또래관계

| 연령대 | 템플릿 예시 |
|---|---|
| 7~9세 | 급식 줄 서기, 쉬는 시간 놀이 참여, 선생님께 도움 요청하기 |
| 10~13세 | 발표하기, 친구에게 먼저 말 걸기, 모둠 활동 참여하기 |
| 14~18세 | 그룹 과제 의견 말하기, 갈등 상황에서 감정 표현하기 |

#### 카테고리 B — 일상생활·감정조절

| 연령대 | 템플릿 예시 |
|---|---|
| 7~9세 | 화가 날 때 멈추기, 물건 정리하기, 새로운 장소 가기 |
| 10~13세 | 실수했을 때 대처, 차례 기다리기, 루틴 변화 받아들이기 |
| 14~18세 | 스트레스 조절 전략, 사과하고 화해하기, 감정 일기 쓰기 |

---

### F3. AI 스토리 자동생성 — 3-Track 생성 모델

스토리 생성 주체(역할)에 따라 입력 컨텍스트와 AI 프롬프트 파이프라인을 분기한다. 세 Track 모두 Carol Gray 10.4 기준과 6WH 구조를 공통으로 적용하되, **입력 가이드·프롬프트 초점·공유 방향**이 다르다.

---

#### Track A — 치료사 주도 생성

**생성 맥락:** 치료 목표(IEP/치료 계획) 기반. 임상적 판단이 반영된 스토리가 필요한 경우.

**입력 화면 구성:**

```
┌─────────────────────────────────────────────────┐
│  치료 목표 기반 소셜스토리를 만들어요               │
│  ┌─────────────────────────────────────────────┐ │
│  │  (아동의 현재 도전 상황을 자유롭게 적어주세요)   │ │
│  └─────────────────────────────────────────────┘ │
│                                                  │
│  🎯 치료 목표 연결 (선택)                          │
│  [ 사회적 의사소통 ] [ 감정 조절 ] [ 행동 전환 ]    │
│  [ 또래 상호작용 ] [ 일상 루틴 ]                   │
│                                                  │
│  📋 치료사 전용 6WH 가이드                         │
│  👤 누가   치료 대상 아동 + 등장인물               │
│  ⏰ 언제   치료 세션 / 일상 어느 시점?              │
│  📍 어디서  클리닉 / 학교 / 가정 중 어디?           │
│  💬 무엇을  현재 어떤 행동·상황이 목표인가요?        │
│  ✋ 어떻게  어떤 대처 행동을 목표로 하나요?          │
│  ❤️ 왜     이 행동이 왜 중요한가요?                │
│                                                  │
│  📊 청킹 전략 선택                                │
│  ( ) 시간적  ( ) 공간적  ( ) 혼합                  │
│                                                  │
│              [ 스토리 만들기 ✨ ]                  │
└─────────────────────────────────────────────────┘
```

**AI 프롬프트 특이사항:**
- 치료 목표 태그를 프롬프트에 자동 주입 → 목표 행동 중심 지시문 배치
- Carol Gray 8th 기준(서술 우선·코칭 최소화) 엄격 적용
- 생성 후 보호자에게 "가정 활용 요청" 알림 발송 가능

---

#### Track B — 보호자 주도 생성

**생성 맥락:** 일상·가정 상황 기반. 아동을 가장 잘 아는 보호자가 직접 상황을 묘사하는 경우.

**입력 화면 구성:**

```
│  ┌─────────────────────────────────────────────┐ │
│  │                                             │ │
│  │  (자유롭게 상황을 적어주세요)                  │ │
│  │                                             │ │
│  └─────────────────────────────────────────────┘ │
│                                                  │
│  ✅ 이런 내용이 들어가면 더 좋은 이야기가 돼요         │
│  ┌─────────────────────────────────────────────┐ │
│  │ 👤 누가   누가 등장하나요? (아이, 친구, 선생님)  │ │
│  │ ⏰ 언제   언제 일어나는 일인가요? (점심시간)     │ │
│  │ 📍 어디서  어디에서 일어나나요? (급식실)         │ │
│  │ 💬 무엇을  어떤 상황인가요? (새치기를 당했어요)   │ │

│  │ ❤️ 왜     왜 그렇게 하면 좋을까요? (모두 기다려요) │ │
│  └─────────────────────────────────────────────┘ │
│                                                  │
│  💡 예시처럼 적어보세요                            │
│  "민준이가(누가) 점심시간에(언제) 급식실에서(어디서)   │
│   줄을 서다 친구가 새치기를 해서 화가 나요(무엇을).    │
│   화를 참고 선생님께 차분히 말하는 법(어떻게)을        │
│   알려주고 싶어요. 모두가 차례를 기다리니까요(왜)."     │
│                                                  │
│              [ 스토리 만들기 ✨ ]                  │
└─────────────────────────────────────────────────┘
```

**AI 프롬프트 특이사항:**
- 보호자 친화적 언어로 6WH 요소 추출 (전문 용어 미사용)
- 생성 후 치료사에게 "검토 요청" 알림 발송 (선택)
- 치료사 미연결 상태에서도 단독 생성·사용 가능

**상황 유형별 예시 문구 (선택 시 입력란에 자동 삽입):**

| 상황 유형 | 예시 문구 (6WH 반영) |
|---|---|
| 또래 갈등 | "지호가(누가) 쉬는 시간에(언제) 교실에서(어디서) 친구가 장난감을 빼앗아 속상해해요(무엇을). 화내지 않고 '돌려줘'라고 말하는 법(어떻게)을 배우면 좋겠어요. 그래야 친구와 사이좋게 지낼 수 있으니까요(왜)." |
| 감정 조절 | "서연이가(누가) 수업 중에(언제) 교실에서(어디서) 답을 틀리면 부끄러워 울어버려요(무엇을). 실수해도 괜찮다고 생각하며 다시 시도하는 법(어떻게)을 알려주고 싶어요. 누구나 실수하며 배우니까요(왜)." |
| 일상 전환 | "준우가(누가) 잠자기 전(언제) 집에서(어디서) 놀이를 멈춰야 할 때 떼를 써요(무엇을). 자기 전 루틴을 자연스럽게 받아들이는 법(어떻게)을 알려주고 싶어요. 푹 자야 다음 날 기분이 좋으니까요(왜)." |
| 새로운 환경 | "하은이가(누가) 다음 주에(언제) 치과에서(어디서) 처음 진료를 받아요(무엇을). 무서워하지 않고 의사 선생님 말을 따르는 법(어떻게)을 미리 준비시키고 싶어요. 건강한 이를 지킬 수 있으니까요(왜)." |

#### 공통 생성 후 편집

- **인라인 편집:** 생성된 각 페이지 텍스트 직접 수정 가능
- **AI 재생성:** 페이지 단위 "다시 생성" 버튼
- **아바타 이미지 교체:** 각 페이지 이미지 변경 가능
- **청킹 방식 전환:** 생성 후에도 시간적/공간적/혼합 청킹 재적용 가능

#### 6WH 작성 가이드 항목 (공통)

| 아이콘 | 항목 | 가이드 질문 | 예시 |
|---|---|---|---|
| 👤 | **누가 (Who)** | 누가 등장하나요? | 민준이, 친구들, 선생님 |
| ⏰ | **언제 (When)** | 언제 일어나는 일인가요? | 점심시간 |
| 📍 | **어디서 (Where)** | 어디에서 일어나나요? | 학교 급식실 |
| 💬 | **무엇을 (What)** | 어떤 상황인가요? | 줄 서다 친구가 새치기를 해서 화가 나요 |
| ✋ | **어떻게 (How)** | 어떻게 행동하면 좋을까요? | 깊게 숨 쉬고 선생님께 차분히 말해요 |
| ❤️ | **왜 (Why)** | 왜 그렇게 하면 좋을까요? | 모두가 차례를 기다리고 있으니까요 |

> **가이드는 강제가 아닙니다.** 모든 항목을 채우지 않아도 스토리를 만들 수 있으며, 빠진 정보는 AI가 후속 질문으로 자연스럽게 보완합니다.

#### AI 생성 방식 (공통)

- **자유 줄글 → LLM 분석 → 소셜 스토리 변환** 파이프라인
- GPT-4o가 줄글에서 6WH 요소를 **자동 추출**하여 Carol Gray 10.4 기준에 맞게 재구성
- 누락된 항목이 있으면 AI가 **자연스러운 후속 질문 1~2개** 제시
- YesChat.ai 방식의 실시간 스트리밍 생성
- 아동 프로필(연령·관심사·친숙 환경)을 프롬프트에 자동 결합

#### 스토리 구성 제한 (MVP)

- 최대 **10페이지**
- 페이지당: 이미지(AI 생성 또는 직접 업로드) + 텍스트(설명문·조망문·지시문)
- 제목 페이지(1) + 본문 페이지(최대 8) + 마무리 페이지(1)

### F5. 협업·승인 플로우

#### 그룹 구조

```
보호자 (그룹 생성 및 최종 승인권 보유)
├── 치료사 (Track A 생성 + 수정 제안 권한)
└── 교사   (Track C 생성 + 수정 제안 권한)
```

- 아동 1명당 그룹 1개 생성
- 보호자가 초대 코드 발급 → 치료사/교사 참여
- 그룹 내 모든 스토리 공유 및 자문 가능

#### Track별 협업 시퀀스

**Track A — 치료사 주도**

```
[치료사]
치료 목표 기반 스토리 생성·편집
        ↓
"가정 활용 요청" 알림 → [보호자]
        ↓
보호자 열람 및 확인
        ↓
[확인 완료] → 가정에서 아동과 함께 읽기 시작
[수정 요청] → 피드백 댓글 → 치료사 재수정
```

**Track B — 보호자 주도**

```
[보호자]
일상 상황 기반 스토리 생성·편집
        ↓
"치료사 검토 요청" 발송 (선택)
        ↓
[치료사] 수정 사항 작성 후 "승인 요청" 전송
        ↓
[보호자] 변경 전/후 비교 화면 확인
        ↓
[승인] → 스토리 즉시 반영 + 치료사에게 완료 알림
[거절] → 원본 유지 + 피드백 댓글
```

**Track C — 교사 주도**

```
[교사]
학교 맥락 기반 스토리 생성·편집
        ↓
그룹 전체(보호자·치료사) 자동 공유 알림
        ↓
보호자·치료사 열람 및 자문 댓글 작성
        ↓
[보호자] 가정 연계 확인 완료 표시
[치료사] 치료 목표와 정합성 검토 → 수정 제안 가능
        ↓
교사가 수정 제안 수락 시 → 보호자 최종 승인
```

#### 자문 댓글 구조

- **스토리 레벨:** 스토리 전체에 대한 의견
- **페이지 레벨:** 특정 페이지 옆 말풍선 아이콘 탭 → 해당 페이지 댓글 작성
- 댓글 작성자 표시 (이름 + 역할 배지)
- 실시간 읽음 확인 (✓ / ✓✓)
- 알림: 앱 내 배지 + 이메일 알림
- 승인 히스토리 영구 보관

## 6. Carol Gray 10.4 기준 반영

| 기준 번호 | 기준명 | MVP 반영 방식 |
|---|---|---|
| **1st** | 하나의 철학·정의·목표 | Social Humility 철학 기반, "내 이야기" 1인칭 서술 고정 |
| **2nd** | 발견: 스토리 또는 대안적 해결책? | 자유 줄글 분석 후 AI가 스토리 생성 또는 대안 개입 제안 여부 판단 |
| **3rd** | 구조: 조직화 및 서술 | 도입(1p) + 본문(최대 8p) + 결론(1p) 3파트 자동 생성 |
| **4th** | 형식: 맞춤화 및 개인화 | 아동 프로필(연령·관심사·친숙 환경) → AI 프롬프트 자동 주입 |
| **5th** | 어조: 안전 및 존중 | 긍정적·인내심 있는 어조 프롬프트 고정, 1인칭/3인칭만 사용 |
| **6th** | 6가지 질문: 고려 및 답변 | Track별 6WH 작성 가이드 제공, AI가 요소 추출 후 누락 시 후속 질문으로 보완 |
| **7th** | 축하: 칭찬 및 긍정 | 스토리 전체 문장의 최소 50% 이상 긍정·칭찬 문장 AI 조건 설정 |
| **8th** | 공식: 서술 우선·코칭 최소화 | 서술문 : 지시문 = 4:1 비율 AI 프롬프트 강제 |
| **9th** | 수정: 검토 및 교정 | 생성 후 인라인 편집, AI 재생성, 전문가 자문 댓글 |
| **10th** | 공유: 소개 및 모니터링 | 3-Track 협업 공간에서 공유, 그룹 내 모니터링, 승인 히스토리 |

## 7. 문장 유형 구조

Carol Gray 소셜 스토리의 핵심 3가지 문장 유형을 각 페이지에 자동 적용.

### 문장 유형 정의

| 유형 | 정의 | 예시 |
|---|---|---|
| **설명문 (Descriptive)** | 상황에 대한 객관적 사실을 서술 | "급식 시간에 친구들은 줄을 서서 밥을 받아요." |
| **조망문 (Perspective)** | 타인의 감정·생각·의도를 묘사 | "선생님은 모든 친구들이 공평하게 기다려 주기를 원해요." |
| **지시문 (Coaching)** | 아동이 할 수 있는 대처 행동 제시 | "나는 숨을 한 번 크게 쉬고 차례를 기다릴 수 있어요." |

### 페이지별 구성 예시

```
[페이지 3 예시 — 급식 줄 서기]

🖼️ [아바타 이미지: 민준이가 줄을 서는 모습]

📌 설명문
"점심시간에 급식실에서 친구들은 차례로 줄을 서서 밥을 받아요."

💭 조망문
"선생님과 친구들은 모두가 기다리면 더 빨리 밥을 먹을 수 있다고 생각해요."

✅ 지시문
"나는 줄 맨 뒤에 서서 내 차례를 기다릴 수 있어요."
```

### AI 프롬프트 비율 규칙

```
설명문 + 조망문 : 지시문 = 최소 4 : 1
- 10페이지 기준: 지시문 포함 페이지는 최대 2페이지
- 나머지 페이지는 설명문 또는 조망문 중심
- 전체 문장의 최소 50% 는 긍정·칭찬 내용
```

## 8. 시공간적 청킹 전략

인지부하이론 및 상황 모델 이론에 근거한 세 가지 정보 제시 방식.

| 청킹 유형 | 설명 | 표지어 예시 | 권장 대상 |
|---|---|---|---|
| **시간적 청킹** | 사건을 시간 순서에 따라 분절 | 먼저 → 그 다음 → 마지막으로 | 순서 이해 훈련 필요 아동 |
| **공간적 청킹** | 장소 맥락에 따라 정보 분절 | 교실에서 → 복도에서 → 급식실에서 | 환경 전환 어려움 아동 |
| **혼합 청킹** | 시간 + 공간 표지어 동시 사용 | 급식실에서, 먼저 → 그 다음 | 실행 기능 향상 목표 |

스토리 생성 시 청킹 유형은 **보호자·치료사·교사 모두** 직접 선택하거나, 아동 프로필 기반으로 AI가 추천.

## 10. 협업·승인 플로우

### 공통 규칙

- 모든 스토리의 **최종 승인권은 보호자**에게 있음
- 치료사·교사가 **본인이 생성한 스토리를 수정**하는 경우 → 보호자 승인 불필요
- 치료사·교사가 **타인(보호자 포함) 생성 스토리를 수정** 제안하는 경우 → 보호자 승인 필요
- 모든 수정 이력은 **승인 히스토리**에 영구 보관
- 생성 주체(역할·Track)는 스토리 메타데이터에 기록되어 뷰어 상단에 표시

### 상세 시퀀스

Track별 시퀀스는 F5 협업·승인 플로우 섹션 참조.

```
공통 단계 요약:
1. 생성 주체(치료사/보호자/교사)가 스토리 완성 후 저장
      ↓
2. Track에 따라 자동 알림 발송 (알림 매트릭스 참조)
      ↓
3. 수신자 열람 → 자문 댓글 또는 수정 제안
      ↓
4-A. [승인] → 스토리 즉시 반영 + 요청자에게 완료 알림
4-B. [거절] → 원본 유지 + 피드백 댓글 → 재수정 가능
      ↓
5. 전체 이력 "승인 히스토리" 탭에 영구 기록
```

### 알림 매트릭스

| 이벤트 | 보호자 | 치료사 | 교사 |
|---|---|---|---|
| Track A 스토리 생성 완료 | 🔔 가정 활용 요청 | — | — |
| Track B 검토 요청 발송 | — | 🔔 검토 요청 | — |
| Track C 스토리 공유 | 🔔 새 스토리 | 🔔 새 스토리 | — |
| 수정 제안 전송 | 🔔 승인 요청 | — | — |
| 보호자 승인 완료 | — | 🔔 완료 | 🔔 완료 |
| 보호자 거절 | — | 🔔 거절+댓글 | 🔔 거절+댓글 |
